## Summary
Testing on a real iPhone showed magic link sign in failing on Safari iOS. The
magic link opens in a different browser context than the one that requested it,
so the Supabase session cookie is not present and the user lands logged out. It
works on desktop Chrome and Android.

## What was done
- Reproduced on a physical iPhone (Safari) three times.
- Confirmed the link opened in the in app browser or a fresh Safari context,
  detached from the requesting tab.
- Confirmed desktop and Android are unaffected.
- Ruled out Supabase config: the same link works when opened in the original
  context on desktop.

## Decisions made
- None yet. This is the problem statement that later drove the OTP fallback.

## Pending
- Design a sign in path that completes in the same tab (no cross context link).
- This is the second time a Safari iOS quirk has bitten us; watch for a pattern.
