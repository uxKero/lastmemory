## Summary
Built the first auth flow: passwordless sign in using Supabase magic links, and
nothing else. User types email, gets a link, clicks it, lands signed in. Session
kept in a Supabase cookie refreshed in middleware.

## What was done
- Sign in page with a single email field.
- Called `supabase.auth.signInWithOtp({ email })` in magic link mode.
- Added `src/middleware.ts` to refresh the Supabase session on every request.
- Redirect target `/invoices` after successful sign in.

## Decisions made
- Magic links as the only sign in method, to keep the auth surface tiny at
  launch. See [[dec-0003]] (later superseded).

## Pending
- Test sign in on real mobile devices, especially iOS Safari.
- Consider a fallback if links prove unreliable.
