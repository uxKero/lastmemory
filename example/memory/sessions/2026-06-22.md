## Summary
Built the billing surface: invoice list, editor and detail view. Hit a build
break: `@react-pdf/renderer` imported into a Server Component passed `next dev`
but failed `next build` on Vercel. Moved PDF rendering into a Route Handler.
Added indexes to speed the invoice list.

## What was done
- `src/app/invoices/page.tsx` (list), the editor and the detail view.
- Line items in integer centavos ARS with a live total.
- First tried rendering the invoice PDF in the Server Component page; `next dev`
  worked, `next build` failed on Vercel with a browser only dependency error.
- Moved PDF to `app/invoices/[id]/pdf/route.ts`, rendered at request time.
- Added index `invoices(user_id, status, created_at)` via a new migration.

## Decisions made
- No new ADR; billing UI inherits [[dec-0001]] and [[dec-0002]].

## Scars
- `@react-pdf/renderer` breaks the Next.js 14 build from a Server Component; keep
  it in a Route Handler or a client boundary. See [[scar-react-pdf-build]].

## Pending
- Disable the editor hard on paid invoices (immutability), not just hide save.

## Full log
- Spent most of the afternoon on the build break because it only failed in the
  production build, not in dev. Lesson: for bundling issues, reproduce with
  `next build` locally, never trust `next dev`.
- Considered `puppeteer` HTML to PDF instead; rejected as too heavy for a
  serverless function and slow cold starts.
