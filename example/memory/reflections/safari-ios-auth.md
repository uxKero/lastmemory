## Summary
Recurring pattern: Safari iOS breaks any auth flow that depends on returning to
the originating browser context. Three separate sessions hit the same wall, and
each time the root cause was cross context navigation, not a Supabase bug or a
misconfiguration. The durable lesson is a design rule, not a one off fix.

## Pattern
- [[session-2026-05-28]]: shipped magic link only. The flow depends on the link
  reopening the requesting context.
- [[session-2026-06-15]]: found magic links fail on Safari iOS because the link
  opens in a different context and the session cookie is missing.
- [[session-2026-07-01]]: fixed it by moving to a same tab OTP code, which never
  leaves the originating context.

Two of the three were framed at the time as "a magic link bug." Seen together
they are one incompatibility: Safari iOS does not reliably preserve session
state across a context switch.

## Durable lesson
Design Orbit auth (and any future auth like flow, for example email confirmation
or re auth) to complete in the same tab it started in. Treat any flow that hands
off to an external link and expects to come back as broken on Safari iOS until
proven otherwise. This rule is now encoded in [[zone-auth]] under Watch out and
realized by [[dec-0004]].
