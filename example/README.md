# Orbit: a sample lastmemory network

This folder is a complete, ready to browse example of a `lastmemory` memory
network. The fictional project is **Orbit**, a SaaS invoicing app for
freelancers built on Next.js, Supabase (auth and Postgres), MercadoPago
(payments) and Resend (email).

It exists for two reasons: it is the visual demo (the graph is generated from
these files) and it is the benchmark scenario. Everything under `memory/` is
interconnected on purpose, so the rendered graph shows real synapses between
neurons rather than a scatter of dots.

## Open it

Read `memory/BRAIN.md` first. It is the map of content (the MOC): it lists every
zone, decision, scar, reflection and recent session with its temperature, and a
"Pick up here" note. From there, follow the links.

## Regenerate the graph

The self contained, offline `graph.html` is produced from these markdown files
with the graph script:

```
python scripts/generate_graph.py example/memory --out example/memory/graph.html
```

Open the resulting `graph.html` in any browser. Nodes are colored by neuron
type, sized by connection count and faded by temperature; edges are colored by
relation type. The one orphan node appears with a dashed border.

## One of each neuron type to look at

- **Session**: `memory/sessions/2026-07-01.md` (full level, shipped the OTP
  fallback and superseded an earlier decision).
- **Zone**: `memory/zones/auth.md` (the hot zone, links to its active and
  superseded decisions and to the reflection).
- **Decision**: `memory/decisions/0004-otp-email-fallback.md` (ADR that
  supersedes `0003-magic-links-only.md`, which is kept as `superseded` with its
  text rewritten in past tense).
- **Scar**: `memory/scars/react-pdf-build-break.md` (high confidence, with
  explicit scope, an expiry hint and a "How to re-validate" section).
- **Reflection**: `memory/reflections/safari-ios-auth.md` (generated by a dream,
  citing three sessions in `source_ids`).

## Deliberate orphan

`memory/sessions/2026-05-05.md` (repo scaffolding) has no links on purpose, so
the graph and the health check have something to flag. Everything else is
connected.
